## Exercise 2: E-commerce Platform Search Function

Big O notation helps in representing the time or space complexity of an algorithm.
It helps us to compare the efficiency of the algorithms. This is done by taking the size of the input as an argument (n).

### Linear Search

In this we go through the entire array once.

Best Case: O(1) - The target element was the first element.

Average Case: O(n)

Worst Case: O(n) - The target element was at the end of the array.

### Binary Search

We need to sort the array before doing the search. In this we divide the array into half as we go.

Best Case: O(1) - The target element was the middle element.

Average Case: O(log n)

Worst Case: O(log n) - The target element was at the extremes of the array.
